Tipos primitivos en JavaScript

Los tipos primitivos representan pequeñas piezas de información como por ejemplo true ó 25. Existen 5 tipos primitivos en JavaScript:

Boolean
true, false
Number
Cualquier valor numérico ya sea entero o de coma flotante
String
Un carácter o secuencia de caracteres delimitado por comillas simples o dobles
Null
Un tipo primitivo que tiene un único valor, null
Undefined
Un tipo primitivo que tiene un único valor, undefined. Es el valor asignado a una variable que no está inicializada.

Los tres primeros tipos (Boolean, Number y String) se comportan de forma similar, mientras que los dos últimos (null y undefined funcionan de una forma un poco distinta. Todos los tipos primitivos tienen una representación literal de su valor. Los literales representan valores: "Aitor", 25, 1.51, true, null, undefined.

En JavaScript una variable que contiene un tipo primitivo contiene directamente ese valor, no un puntero hacia él). Cuando asignas un valor primitivo a una variable, el valor se copia en esa variable. Esto significa que si estableces una variable igual a otra, cada variable contiene su propia copia del dato. Por ejemplo:

            
var color1 = "rojo";
var color2 = color1;

Aquí, el valor "rojo" se asigna a la variable color1 y posteriormente a la variable color2 se le asigna el valor almacenado en color1, que es "rojo". A pesar de que color1 y color2 contienen el mismo valor, son variables independientes una de otra. Puedes cambiar el valor de una ellas sin afectar a la otra. Como las dos variables contienen valores de tipo primitivo, usan su propio alojamiento separado en memoria y los cambios en una variable no se ven reflejados en la otra.


var color1 = "rojo";
var color2 = color1;

console.log(color1); // "rojo"
console.log(color2); // "rojo"

color1 = "azul";

console.log(color1); // "azul"
console.log(color2); // "rojo"

En el código anterior, color1 se cambia a "azul" y color2 no se ve afectada y sigue manteniendo "rojo".

Identificando tipos primitivos

La mejor forma de identificar tipos primitivos es utilizando el operador typeof, que funciona sobre cualquier tipo de variable y devuelve un string indicando el tipo de dato que contiene. A continuación unos ejemplos de su uso:


console.log(typeof "Aitor"); // "string"
console.log(typeof 25); // "number"
console.log(typeof 3.7); // "number"
console.log(typeof false); // "boolean"
console.log(typeof undefined); // "undefined"

Hasta aquí todo normal y con resultados como cabría esperar. La parte extraña viene cuando utilizamos typeof con null.


console.log(typeof null); // "object"

Sí, devuelve "object". Pero ¿por qué "object" cuando en realidad el tipo sería null?. Se podría argumentar que null es un puntero vacío hacia un objeto. Aún así sigue siendo confuso.

La mejor manera es comprobar si un valor es null es compararlo directamente con null.


console.log(valor === null); // true o false

Métodos primitivos

A pesar de ser tipos primitivos (no son objetos) tienen métodos built-in. null y undefined no tienen métodos. Algunos ejemplos:


var nombre = "Aitor";
var nombreMinuscula = nombre.toLowerCase();
var primeraLetra = nombre.charAt(0);
var precio = 10;
var descripcionPrecio = precio.toFixed(2); // "10.00"
var todoOK = true;
console.log(todoOK.toString()); // "true"
